# Ray 1.0.0 General Availability Release Notes

We are thrilled to announce the official General Availability release of **Ray 1.0.0** — the high-performance, modular build system and toolchain for modern web applications!

---

## Key Highlights

- **Native Rust Compiler Engine**: Optional Rust backend (`@ray/compiler-rust`) delivering ultra-fast tokenization and tree-shaking with zero-cost TypeScript fallbacks.
- **First-Class Multi-Framework Support**: Official plugins for **React** (including RSC and Server Actions), **Vue 3**, **Svelte**, **SolidJS**, **Angular**, and **MDX**.
- **Edge Runtime Foundation & Deployment Framework**: Target Web-standard Edge runtimes (`--target edge`) and provider-agnostic deployment adapters (`@ray/deployment`).
- **Comprehensive Workspace & Task Orchestrator**: Project graph dependency analysis (`ray graph`), affected project detection (`ray affected`), and distributed remote caching (`@ray/observability` & `@ray/inspector`).
- **Native Test Engine & VS Code Extension**: Built-in Vitest/Jest compatible test runner (`ray test`) and active VS Code extension (`@ray/vscode-extension`).

## Migration Guides

### Migrating from Vite
1. Replace `vite.config.ts` with `ray.config.ts`:
```typescript
import { defineConfig } from '@ray/core';
import vue from '@ray/plugin-vue';

export default defineConfig({
  plugins: [vue()],
});
```
2. Update CLI scripts in `package.json`:
   - `vite` -> `ray dev`
   - `vite build` -> `ray build`

### Migrating from Webpack / Parcel / Rollup
- Ray natively resolves ES modules, CSS Modules, Tailwind, Sass, and PostCSS out of the box without requiring complex loader chains.

## Semantic Versioning & Support Policy

- **Ray Core & Official Plugins**: Strict Semantic Versioning (`MAJOR.MINOR.PATCH`).
